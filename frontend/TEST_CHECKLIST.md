# AVISHU Frontend Test Checklist

## Вход и маршрутизация

- Вход под `client@avishu.com` открывает экран клиента
- Вход под `franchisee@avishu.com` открывает экран франчайзи
- Вход под `production.manager@avishu.com` открывает экран менеджера цеха
- Вход под `worker.cutting@avishu.com` открывает экран сотрудника цеха
- Выход возвращает на экран входа

## Клиент

- Баннер коллекции и каталог товаров отображаются
- Для товара `in_stock` доступна покупка
- Для товара `made_to_order` доступен предзаказ с выбором даты
- Новый заказ появляется в блоке клиента со статусом `placed`
- После mock payment заказ переходит в `paid`
- Трекинг заказа меняется по цепочке `placed → paid → accepted → in_production → ready`
- Блок лояльности отображается на клиентском профиле

## Франчайзи

- Новый заказ появляется без перезапуска приложения
- Дашборд показывает метрики-заглушки
- Заказы разложены по канбан-секциям
- Доступен переход `placed → accepted`
- Доступен переход `accepted → in_production`
- `ready`-заказы не предлагают лишних действий

## Производство

- После перевода заказа в цех создаются 4 этапа: `cutting`, `sewing`, `finishing`, `qc`
- Менеджер видит всю очередь по франшизе и список сотрудников
- Менеджер может назначить исполнителя только с совпадающей специализацией
- Менеджер может отфильтровать задачи по статусу, сотруднику и этапу
- Сотрудник видит только свои задачи
- Для `assigned` доступна кнопка `Взять в работу`
- Для `in_progress` доступна крупная кнопка завершения
- После завершения финального этапа `qc` заказ получает статус `ready`

## Сервер и синхронизация

- Вся основная цепочка работает без mock-режима
- Логин, товары, заказы и production-задачи берутся с backend
- Мгновенные обновления приходят через `/ws`
- Резервный опрос не ломает основной realtime-поток
- Явный mock-режим включается только через `USE_MOCK=true`

## Состояния

- Во время входа показывается состояние загрузки
- Пустой каталог и пустые заказы показывают отдельные empty-state
- Ошибки API показывают retry-state
- Кнопки действий отключаются на время запроса
